Report Generation Methodology

The Sunsynk Report Platform generates comprehensive system performance analyses using real-time telemetry data from your inverter. All metrics are calculated using rigorous mathematical methods with clear assumptions and validation rules.

This knowledge base documents the exact formulae, data processing steps, and quality assurance thresholds applied to your system data.

1. Energy at a Glance (Cumulative Meter Totals)

All six primary energy metrics are computed as delta readings from the inverter's on-device lifetime odometer counters.

Metric Source Field Formula
Solar Generated pvHistory max(pvHistory) − min(pvHistory)
Home Energy Used totalUsed max(totalUsed) − min(totalUsed)
Grid Import etotal_from max(etotal_from) − min(etotal_from)
Grid Export gridSellTotal max(gridSellTotal) − min(gridSellTotal)
Battery Charged batChargeTotal max(batChargeTotal) − min(batChargeTotal)
Battery Discharged batDischargeTotal max(batDischargeTotal) − min(batDischargeTotal)

All values are expressed in kWh. A metric is flagged data-gap when both its start and end readings equal zero.

2. Reconciled Energy Flow (Interval Integration)

This section converts instantaneous power readings into cumulative energy using trapezoidal approximation (rectangular integration with forward-fill).

2.1 Interval Energy from Power

For each telemetry interval [ti, ti+1]:

$$\Delta t = t_{i+1} - t_i \quad \text{(hours)}$$ $$E = \frac{\max(P_W,\ 0) \times \Delta t}{1000} \quad \text{(kWh)}$$

An interval is excluded from energy integration when:

  • Any of the four power signals (PV, load, grid, battery) is null
  • Δt ≤ 0 or Δt > 20 minutes (contiguous gap threshold)

Sign conventions:

  • Grid: positive = import, negative = export → Eimport = max(Pgrid, 0), Eexport = max(−Pgrid, 0)
  • Battery: positive = discharge, negative = charge → Edis = max(Pbat, 0), Echg = max(−Pbat, 0)

2.2 Energy Flow Path Allocation (Per Interval)

Each interval's energy is decomposed into seven flow paths using a greedy priority cascade:

  1. pvToLoad = min(Epv, Eload)
  2. remainLoad = Eload − pvToLoad
  3. batteryToLoad = min(Edis, remainLoad)
  4. remainLoad = remainLoad − batteryToLoad
  5. gridToLoad = min(Eimport, remainLoad)
  6. pvToBattery = min(Echg, remainPv)
  7. gridToBattery = min(Eimport, Echg)

Paths are accumulated over all eligible intervals and rounded to 2 decimal places.

2.3 Residuals (Data Quality Indicator)

$$\text{sourceExcess} = \max\bigl((E_{pv} + E_{import} + E_{dis}) - (E_{load} + E_{chg} + E_{export}),\ 0\bigr)$$ $$\text{sinkExcess} = \max\bigl((E_{load} + E_{chg} + E_{export}) - (E_{pv} + E_{import} + E_{dis}),\ 0\bigr)$$

Non-zero residuals indicate measurement imbalance (sensor drift or firmware rounding).

2.4 Coverage

$$\text{coveragePercent} = \frac{\text{eligibleIntervals}}{\text{totalIntervals}} \times 100$$

3. Solar Utilisation

3.1 Self-Sufficiency

$$\text{Self-Sufficiency}\ (\%) = \frac{E_{load} - E_{import}}{E_{load}} \times 100$$

Equivalently: 1 − (Eimport / Eload). Derived from reconciled flow paths:

$$= \frac{pvToLoad + batteryToLoad}{E_{load}} \times 100$$

Thresholds: ≥ 70% = High (green), ≥ 40% = Moderate (amber), < 40% = Low (red).

3.2 PV / Load Ratio

$$\text{PV/Load Ratio} = \frac{E_{pv}}{E_{load}}$$

3.3 Overproduction

A day is overproducing when Ppv > Pload whilst SOC ≥ 99%.

$$\text{ExcessPower}_W = \max(P_{pv} - P_{load},\ 0)$$ $$\text{UnusedPotential}_{kWh} = \sum_{\text{intervals}} \frac{\text{ExcessPower}_W \times \Delta t_{\text{seconds}}}{3\,600 \times 1\,000}$$ $$\text{OverproductionPct}\ (\%) = \frac{\text{overproductionDays}}{\text{totalDays}} \times 100$$

4. Inverter Clipping

Detects when PV output is constrained at the inverter AC rating ceiling.

4.1 Clipping Threshold

$$P_{\text{clip}} = P_{\text{rated}} \times 0.98 \quad \text{(kW)}$$

A sample is a candidate clip when:

  • Ppv ≥ Pclip
  • Within the daylight window 05:00–20:00
  • Gap to next reading ≤ 20 minutes

4.2 Event Qualification

Consecutive candidate-clip samples are grouped into events. An event is qualifying only if:

  • intervals ≥ 3 AND
  • duration ≥ 15 minutes

4.3 Summary Metrics

$$\text{ClippedDurationHours} = \sum_{\text{qualifying events}} \text{event duration (hours)}$$ $$\text{ClippingThreshold}_{kW} = \frac{P_{\text{rated}} \times 0.98}{1000}$$

5. System Accuracy Check

Validates internal consistency of the inverter's power balance reporting.

5.1 Expected Load

$$P_{\text{expected}} = P_{ac} + P_{inv}$$

5.2 Inconsistency Flag (Per Sample)

A sample is flagged inconsistent when:

$$|P_{\text{expected}} - P_{load}| > \max(50\ \text{W},\ P_{load} \times 0.05)$$

5.3 Low-Load Exclusion

A sample is marked low-load (excluded from the real inconsistency rate) when:

$$P_{load} < 50\ \text{W} \quad \text{OR} \quad P_{\text{expected}} < 50\ \text{W}$$

5.4 Real Inconsistency Rate (Per Day)

$$\text{RealInconsistencyPct}\ (\%) = \frac{\text{realInconsistentRows}}{\text{eligibleRows}} \times 100$$

where eligibleRows = all rows with lowLoadFlag = 0.

Thresholds: warning triggered at ≥ 10% real inconsistency with ≥ 3 inconsistent rows.

6. Battery Health

6.1 Daily Derived Metrics

Metric Formula
Cycle Depth CycleDepth (%) = SOCmax − SOCmin
Charge/Discharge Ratio rcd = ∑Pchg / ∑Pdis
Grid Dependency Ratio rgrid = ∑Pgrid / ∑Pload
Charge-Grid Ratio rcg = ∑Pchg / ∑Pgrid
Temperature Exposure rtemp = Nhot / Ntotal where hot = Tbat > 35°C
Total Cycle Count Ctotal = Edis,cumulative / Ecapacity,kWh
Daily Cycle Count Cday = Ctotal(d) − Ctotal(d−1)

6.2 Flag Conditions

Flag Condition
UNDERUTILISED_BATTERY rgrid > 0.7 AND SOCmin > 70% AND rcg < 0.3
HIGH_TEMP_EXPOSURE rtemp > 0.3
HIGH_CYCLING_STRESS CycleDepth > 60%
HIGH_DISCHARGE_STRESS rcd < 0.7
LOW_UTILIZATION_OR_LOSS rcd > 1.3
BALANCED 0.7 ≤ rcd ≤ 1.3

6.3 Data Confidence

Level Condition
high ≥ 30 days AND ≥ 10,000 samples
medium ≥ 14 days AND ≥ 2,000 samples
low ≥ 7 days AND ≥ 100 samples
insufficient < 7 days OR < 100 samples

7. Night-Time Grid Dependency (Evening Efficiency)

All calculations are scoped to the night-time window (configurable; default 18:00–23:00).

7.1 Daily Ratios

Metric Formula
Grid Dependency Ratio rgrid = ∑Pgrid / ∑Pload
Self-Sufficiency Ratio rss = 1 − rgrid
Battery Contribution Ratio rbat = ∑Pdis / ∑Pload
PV Contribution Ratio rpv = ∑Ppv / ∑Pload
Load Spike Factor rspike = Pload,max / P̄load

7.2 Flag Conditions

Flag Condition
is_high_grid_dependency rgrid > 0.4
is_battery_underutilised SOC̄ > 50% AND rbat < 0.2
is_low_pv P̄pv < 50 W
is_load_spike rspike > 3.0
is_battery_empty_peak count(SOC = 0 AND Pgrid > 0) > 0
grid_loss_event Pgrid = 0 AND Pac = 0 AND Pbat ≥ 0 AND Ppv ≤ 0

7.3 Daily Efficiency Score

$$\text{EfficiencyScore} = 100 - 30 \cdot \mathbf{1}[\text{highGrid}] - 20 \cdot \mathbf{1}[\text{batUnder}] - 10 \cdot \mathbf{1}[\text{loadSpike}] - 40 \cdot \mathbf{1}[\text{batEmpty}]$$

Range: 0–100. Maximum penalty (all flags) = 0.

7.4 Period Average Efficiency Score

$$\overline{\text{EfficiencyScore}} = \frac{1}{N} \sum_{d=1}^{N} \text{EfficiencyScore}_d$$

Rounded to 1 decimal place.

8. Peer Benchmark

8.1 Haversine Distance (Used to Select Peers)

$$d = 2R \arcsin\!\sqrt{\sin^2\!\frac{\Delta\phi}{2} + \cos\phi_1\cos\phi_2\sin^2\!\frac{\Delta\lambda}{2}}$$

where R = 6371 km, φ = latitude, λ = longitude.

Peers must be within 50 km.

8.2 Specific Yield

$$Y = \frac{E_{pv,kWh}}{P_{installed,kWp}} \quad \text{(kWh/kWp)}$$

Specific yield is used for ranking when ≥ 5 peers have capacity data. Otherwise raw Epv is used.

8.3 Rank

$$\text{Rank} = N_{\text{peers with better value}} + 1$$

For example, Rank 1 = best performer in the local cohort.

9. Daily Aggregates (Trend Charts)

Metric Formula
Avg PV Power P̄pv = AVG(Ppv,W) / 1000 (kW)
Peak PV Power Ppv,max = MAX(Ppv,W) / 1000 (kW)
Avg Load P̄load = AVG(Pload,W) / 1000 (kW)
Avg Battery Power P̄bat = AVG(Pbat,W) / 1000 (kW) (positive = discharge)

10. Forward-Fill (LOCF): Data Gap Handling

All time-series queries use Last Observation Carried Forward to fill gaps in 5-minute telemetry. A gap is carried forward only when the next observation is within 20 minutes. Intervals exceeding this gap are excluded from energy integration (Section 2).

Rounding Rules

All energy totals are rounded to 2 decimal places (round2). Efficiency scores are rounded to 1 decimal place. Ranked values and distances are rounded to 1 decimal place.
